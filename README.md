## Description
NetPractice is a practical, introductory project focused on the core fundamentals of computer
networking. The goal of this project is to successfully resolve networking anomalies across 
10 distinct, simulated network environments. By configuring IP addresses, adjusting subnet 
masks, and determining correct default gateways, this project builds a robust understanding 
of how data routing operates between hosts and routers.

## Instructions

### How to run the Training Interface
To launch the simulated training interface, follow these steps in your terminal:

1.	**Automated Launch:** Run the provided shell script from the project folder:
```bash
   ./run.sh
```

2.	**Manual Fallback:** If the shell script fails due to browser security or design 
constraints, manually start a local Python web server:
	
	python3 -m http.server 49242

	Then, navigate to http://localhost:49242 in your preferred web browser.

### How to Export Configurations
Once you successfully complete a level in the interface, a **"Next Level"** button will appear. 
Before moving forward, you must click the **"Get my config"** button to export your solved 
configuration file. Repeat this process for every level to ensure you save your progress.

### Submission Requirements
To successfully submit the project, you must place all **10 exported configuration files**
(one per level) directly at the root of your Git repository. Ensure you have entered your 
correct intranet login before exporting, as the configuration must match your profile for 
evaluation.


## Resources

### Core Networking Concepts Studied
Throughout this project, a variety of fundamental networking principles were analyzed and 
configured, including:
* **TCP/IP Addressing & Subnet Masks:** Calculating valid host ranges, network addresses, 
and understanding CIDR notation versus standard masking.
* **Routers, Switches, & Default Gateways:** Figuring out how data packets transit through 
interconnected nodes, defining routing tables, and mapping default paths out of a local network.
* **The OSI Model & Network Layers:** Visualizing how the network layer operates to achieve 
end-to-end communication between distant hosts.
